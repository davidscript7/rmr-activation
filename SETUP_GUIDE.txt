# GIT + GITHUB SETUP GUIDE
# RMR Activation — davidscript7
# ============================================================

==============================================================
  PART 1 — INSTALL GIT ON WINDOWS
==============================================================

1. Go to: https://git-scm.com/download/win
2. The download starts automatically — run the installer
3. During installation, leave ALL default settings as-is
   (just click Next → Next → Install → Finish)
4. Verify it worked: open any folder, right-click →
   you should see "Git Bash Here" or "Open Git Bash"

--------------------------------------------------------------
  PART 2 — CONFIGURE GIT (do this once)
--------------------------------------------------------------

Open Git Bash (right-click anywhere → Git Bash Here) and run:

  git config --global user.name "Héctor Gallego"
  git config --global user.email "your-github-email@gmail.com"

  (use the same email you registered on GitHub)

--------------------------------------------------------------
  PART 3 — CREATE THE GITHUB REPOSITORY
--------------------------------------------------------------

1. Go to: https://github.com/new
2. Fill in:
     Repository name:  rmr-activation
     Description:      Daily RMR Activation process automation using Python, pandas and SAP Fiori integration
     Visibility:       ✅ Private   ← IMPORTANT
3. Do NOT check "Add a README file" (we already have one)
4. Click "Create repository"
5. GitHub will show you a page with setup instructions —
   you can ignore those, follow this guide instead

--------------------------------------------------------------
  PART 4 — UPLOAD THE PROJECT FILES
--------------------------------------------------------------

Step 4.1 — Download the project ZIP from Claude
  - Download the ZIP file Claude provided
  - Extract it to: C:\Users\[your user]\rmr-activation
    (or wherever you want to keep it permanently)

Step 4.2 — Open Git Bash inside the project folder
  - Open the rmr-activation folder in Windows Explorer
  - Right-click inside the folder → "Git Bash Here"

Step 4.3 — Run these commands ONE BY ONE:

  git init
  git add .
  git commit -m "Initial commit: RMR Activation automation v1.0"
  git branch -M main
  git remote add origin https://github.com/davidscript7/rmr-activation.git
  git push -u origin main

Step 4.4 — GitHub will ask for your credentials:
  - Username: davidscript7
  - Password: use a Personal Access Token (NOT your password)

  How to create a Personal Access Token:
    1. Go to: https://github.com/settings/tokens
    2. Click "Generate new token (classic)"
    3. Note: "rmr-activation push"
    4. Expiration: 90 days (or No expiration)
    5. Scopes: check ✅ repo
    6. Click "Generate token"
    7. COPY the token now (you won't see it again)
    8. Paste it as your password when Git asks

--------------------------------------------------------------
  PART 5 — VERIFY IT WORKED
--------------------------------------------------------------

Go to: https://github.com/davidscript7/rmr-activation

You should see all the files there. The repo will be Private
(only you can see it).

--------------------------------------------------------------
  PART 6 — HOW TO UPLOAD CHANGES IN THE FUTURE
--------------------------------------------------------------

Every time you make changes and want to save a new version:

  git add .
  git commit -m "Brief description of what you changed"
  git push

Examples of good commit messages:
  "Add error handling for missing IW75 columns"
  "Fix CFIN DATE filter to include blank dates"
  "Improve output formatting with color-coded rows"
  "Update config example with new SharePoint structure"

This creates a visible history of improvements —
which looks very professional when you make the repo public.
